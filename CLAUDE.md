техно-андеграунд мерч из Москвы. Бренд пары **Маша и Валера**, тон «не для скромных»,

сам пользователь — **@god_kitty** (Сергей Якимов, me.sergei.kim@gmail.com).

## Структура
```
index.html            — главная (гейт → герой → манифест → дроп «Бэнгер» → лукбук → CTA → футер)

assets/js/site.js     — общие эффекты: кастомный курсор, маркизы, reveal, прогресс, гейт
assets/js/catalog.js  — товары (PRODUCTS), каталог, модалка, корзина, оформление заказа
assets/img|tex|video  — медиа
server.py             — Python-сервер: статика + POST /api/order + телеграм-уведомления
orders.json           — все заказы (массив), создаётся сервером
tg_config.json        — токен бота + белый список ников + chat_id (СЕКРЕТ, не публиковать)
DEPLOY.md             — пошаговый деплой на VPS
deploy/               — sexytips.service (systemd), Caddyfile (HTTPS-прокси)
```

## Как запускать локально
```
python3 server.py        # http://localhost:8000 (или python3 server.py 8001)
```
`?nogate` в URL пропускает интро-гейт (удобно для тестов/скриншотов).

## Система заказов (вся цепочка проверена)
1. В модалке товара кнопка «В КОРЗИНУ» (размер обязателен). Корзина — выезжающая
   панель справа (`#cart`), хранится в `localStorage('st_cart')`, кнопка КОРЗИНА
   в шапке каталога с розовым бейджем-счётчиком на углу.
2. «ПЕРЕЙТИ К ОФОРМЛЕНИЮ»: ИМЯ*, ФАМИЛИЯ*, ТЕЛЕФОН* (валидация), НИК TELEGRAM (опц.),
   СПОСОБ ПОЛУЧЕНИЯ* (личная встреча · Москва / доставка), АДРЕС* только при доставке.
3. «ОФОРМИТЬ» → номер заказа `ST-<base36>` → POST `api/order` → сервер пишет в
   `orders.json` и шлёт уведомление в Telegram. Запасная копия — `localStorage('st_orders')`.
4. Телеграм: бот **@sexxxytipsorders_bot**; уведомления уходят ТОЛЬКО никам из
   `tg_config.json → allowed` (сейчас @god_kitty). chat_id резолвится автоматически
   после того, как человек написал боту /start. Клиентам бот ничего не шлёт —
   «мы сами свяжемся» (так решил пользователь).
5. Сервер НЕ отдаёт наружу: tg_config.json, orders.json, server.py, deploy/, *.md (404).

## Ключевые технические решения (не ломать!)
- **Модалка товара** (десктоп ≥820px): `.modal__card` — flex; ширина фото-колонки —
  чистый CSS: `width:min(calc(min(88vh,840px) * var(--mr)), calc(100vw - 2*clamp(12px,3vw,40px) - 462px))`,
  где `--mr` = аспект текущего фото, ставится JS-функцией `fitMedia()` (единственная
  её задача). Фото заполняет колонку ровно, БЕЗ полос и БЕЗ обрезки — пользователь
  на этом настаивал многократно. Инфо-колонка `flex:1 0 460px;max-width:560px`.
  На ≤1280px — компактный блок (стоит в CSS ПОСЛЕ базовых правил — каскад!).
- **m-fill**: в модалке каждое фото рендерится дважды (размытая подложка + contain),
  подложка почти не видна при точном аспекте — оставлена как страховка.
- **Секретный товар** (id `secret-drop`): карточка в сетке ПОЛНОСТЬЮ невидима,
  проявляется по hover; фото — скрин ТГ-сообщения с подписью про «декс»
  (`secret_drop.jpg`, без интерфейса ТГ). В модалке фото заблюрено до hover.
- **Курсор**: системный скрыт; красное перевёрнутое сердечко остриём в точке клика
  (`.cursor-dot`) + инерционное кольцо (`.cursor-ring`). На интерактивных элементах
  НЕ растёт, только розовеет. z-index 12000/12001 — ВЫШЕ модалки (10500) и корзины (10800).
- **Маркизы**: site.js повторяет набор слов до покрытия ширины окна ×1.2, затем
  зеркалит (анимация -50%); иначе на широких окнах пустота.
- **z-index карта**: fx-оверлеи 9000 < шапка 9100 < прогресс 9500 < модалка 10500 <
  корзина 10800 < гейт 11000 < курсор 12000+.
- **Виньетка/скан-линии** ослаблены (`.fx-vig`, `.fx-scan`) — пользователь жаловался
  на тусклость по краям. Не усиливать.
- Заголовок модалки: `clamp(24px,2.3vw,32px)` — БЕЗ overflow-wrap:anywhere (рвал слова),
  длинное слово «ФИОЛЕТОВАЯ» должно влезать целиком.
- Кеширование: server.py шлёт `Cache-Control: no-cache` для html/css/js (борьба с
  залипанием старых версий), картинки кешируются.

1. **После КАЖДОГО изменения визуально проверять результат** (headless Chrome скриншот
   именно того места, что менял) и только потом говорить «готово». Уже дважды
   обжигались, правило записано и в память Claude.
2. Пользователь не программист — объяснять по-русски, простыми словами, без жаргона.
3. Ничего не должно скачиваться на компьютер клиента сайта; заказы — только в
   серверный файл + телеграм.
4. Частая причина «у меня не работает» — кеш браузера или старый запущенный сервер:
   просить Cmd+Shift+R и перезапуск `server.py`.
5. CSS-ловушки этого проекта: правило позже в файле побеждает (компактный блок модалки
   должен стоять после базовых); `[hidden]` перебивается `display:flex` — для скрываемых
   элементов нужны явные `.x[hidden]{display:none}`.
